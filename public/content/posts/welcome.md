# Welcome to Our New Website!

We're thrilled to announce the launch of our brand new website. This isn't just a fresh coat of paint – it's a completely reimagined way of managing and delivering content.

## What's New?

### No CMS Signup Required

Forget about creating accounts, managing subscriptions, or dealing with complex CMS setups. Our content lives directly in the repository as simple markdown files.

### AI-Friendly Content Creation

Need a new blog post? Just ask! Our AI assistant can create content files directly in your repository. Say something like:

> "Write a blog post about our new product launch"

And watch as the content appears, ready to publish.

### Full Control

Every piece of content is a file in your repository. You can:

- Edit directly in GitHub's web interface
- Clone and edit locally
- Use your favorite text editor
- Track changes with Git
- Review before deploying

## How It Works

Content is stored in two main folders:

```
/content/
  ├── pages/     # Static pages (home, about, contact)
  └── posts/     # Blog posts and articles
```

Each file is a simple markdown document with optional frontmatter for metadata like title, date, and tags.

## Getting Started

1. **Browse the site** – Check out the Home, About, and Blog pages
2. **Edit content** – Modify any markdown file to update the site
3. **Add blog posts** – Create new files in `/content/posts/`
4. **Deploy** – Push to GitHub and Vercel handles the rest
